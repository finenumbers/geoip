## Config store

Все операционные настройки хранятся в Docker volume `config_data` (`/data/geoip/config/`):

| Файл | Содержимое |
|------|------------|
| `settings.json` | Публичные параметры (лимиты, cron flags, CORS) |
| `secrets.enc` | AES-256-GCM: ГРЧЦ creds, API keys, admin hash, session secret, Maps key |
| `proxy.env` | API key для nginx → backend (обновляется при save External key) |
| `meta.json` | Версия, `updatedAt` |
| `.master-key` | Master key (если `CONFIG_MASTER_KEY` не задан в env) |

Bootstrap env (compose): `DATABASE_*`, `CONFIG_DATA_DIR`, опционально `CONFIG_MASTER_KEY`.

Backup: [РАЗВЁРТЫВАНИЕ.md](РАЗВЁРТЫВАНИЕ.md#backup-и-восстановление).

## Первичная настройка

| Шаг | URL / раздел | Обязательно |
|-----|--------------|-------------|
| 1. Admin account | `/admin/setup` | да |
| 2. External IP Lookup key | `/admin/setup-api-key` | **да** |
| 3. ГРЧЦ creds | `?section=grchc` | да |
| 4. Import датасета ГРЧЦ | `?section=overview` | да |
| 5. Import датасета RIR+IANA | `?section=overview` | да |
| 6. Включить автоимпорты + расписание | `?section=general` | да |
| 7. Google Maps | `?section=integrations` | нет |

После шага 1 `/admin/setup` недоступен. Без шага 2 Admin перенаправляет на `/admin/setup-api-key`, data-plane API вернёт **401**.

Checklist на Dashboard и в **Обзор** синхронизирован с `GET /api/v1/public/setup-checklist`.

## Секции Admin UI

Навигация: `/admin?section=<id>`. По умолчанию — **overview**.

### Обзор (`overview`)

| Элемент | Описание |
|---------|----------|
| Setup checklist | Статус 7 шагов onboarding |
| Pending reload | Поля, требующие restart API / reload web |
| **Проверить ГРЧЦ** / **Импортировать ГРЧЦ** | Жёлтая / зелёная кнопки (ряд 1) |
| **Проверить RIR+IANA** / **Импортировать RIR+IANA** | Жёлтая / зелёная кнопки (ряд 2) |
| **Wipe датасетов** | Деструктивная очистка ГРЧЦ+RIR+export history+CC mismatch (`POST /admin/data/wipe`) |
| **Logout** | Выход из session |

### Общие (`general`)

| Поле | Описание | Reload |
|------|----------|--------|
| Часовой пояс | IANA TZ для UI и времени автоимпорта (default `Europe/Moscow`) | runtime |
| Автоимпорт ГРЧЦ + время | Ежедневный cron `geoip_import` (по умолчанию **выключен**) | runtime |
| Автоимпорт RIR+IANA + время | Ежедневный cron `geoip_rir` (по умолчанию **выключен**) | runtime |
| Пропускать неизменённый датасет ГРЧЦ | Skip unchanged fingerprint | runtime |
| Кэш ZIP ГРЧЦ | Кэшировать скачанные ZIP | runtime |

### ГРЧЦ / Import (`grchc`)

**Обязательно.** Личные creds ЛК GeoIP ГРЧЦ.

| Поле | Описание |
|------|----------|
| Email / Пароль | Доступ к личному кабинету GeoIP |
| Base URL | URL ЛК (default `https://geoip.noc.gov.ru`) |
| Staging snapshot | Snapshot staging перед swap |

Расписание автоимпорта и флаги ZIP cache / skip unchanged — в секции **Общие**.

Cron **не запускает** import ГРЧЦ без сохранённых creds.

После save: **Проверить ГРЧЦ** → **Импортировать ГРЧЦ** на Обзоре.

### API и безопасность (`api`)

| Поле | Описание |
|------|----------|
| Ссылка на `/api-docs` | Справка IP API (требует admin-сессию; примеры с реальным ключом) |
| API auth enabled | Требовать `X-API-Key` на data-plane (production default: **true**) |
| **API-ключ External IP Lookup** | Ключ для lookup/table/export/metrics. Задаётся на `/admin/setup-api-key` |
| Import API key | Автогенерируется; **не используется в HTTP** — import только через admin session. См. [ROADMAP.md](ROADMAP.md) §2 |
| CORS origin | Для cross-origin API (default dev: `http://localhost:5173`) |
| Rate limit max / window ms | Лимит запросов на data-plane |

Nginx читает `proxy.env` и подставляет External key в proxied `/api/` — SPA работает без ключа в браузере.

При ротации External key — redeploy `geoip_web` (обновление `proxy.env`).

### Admin доступ (`adminAccess`)

| Поле | Описание |
|------|----------|
| Username | Логин admin |
| Current password | Текущий пароль (обязателен для смены) |
| New password | Новый пароль |

Пароль хранится как scrypt hash в `secrets.enc`.

### Export (`export`)

| Поле | Default | Описание |
|------|---------|----------|
| Retention days | 7 | Срок хранения файлов export |
| Retention limit | 100 | Макс. число jobs |
| Max rows | 5 000 000 | Лимит строк на один export |
| Poll interval ms | 5000 | Интервал опроса export worker |

Export worker формирует CSV (UTF-8 BOM, `;`) в ZIP.

### Производительность (`performance`)

| Поле | Default | Описание |
|------|---------|----------|
| Max page size | 200 | Макс. строк на страницу Browse |
| Max offset page | 500 | Лимит offset-пагинации |
| DB pool max | 20 | Размер пула соединений |
| Statement timeout ms | 30000 | Таймаут SQL |
| ASN batch size | 50000 | Batch ASN mapping |
| ASN workers | 6 | Параллельные workers ASN |

### Интеграции (`integrations`)

| Поле | Описание |
|------|----------|
| Google Maps API key | Maps Embed API для карты на Lookup |

Ключ отдаётся клиенту через `GET /api/v1/public/runtime` — ограничьте по HTTP referrer в GCP. См. [ПЕРИМЕТР-И-HTTPS.md](ПЕРИМЕТР-И-HTTPS.md).

### Логирование (`logging`)

| Поле | Default | Описание |
|------|---------|----------|
| Log level | info | fatal … trace |
| Access log | true | HTTP access log API |

### Инфраструктура (`infra`)

Read-only: пути import/export dir. Postgres, compose, NPM — вне Admin store ([РАЗВЁРТЫВАНИЕ.md](РАЗВЁРТЫВАНИЕ.md)).

## Import

### Ручной запуск

Admin → **Обзор** → **Импортировать датасет ГРЧЦ**

API: `POST /api/v1/admin/imports/trigger` (admin session).

| Ответ | Значение |
|-------|----------|
| 200 `{ ok, importRunId }` | Import поставлен в очередь |
| 409 `ImportAlreadyRunning` | Import уже выполняется |

### Автоimport (cron)

Настраивается в Admin → **Общие**:

| Поле | Default |
|------|---------|
| Часовой пояс | Europe/Moscow — также TZ для времени автоимпорта |
| Автоимпорт ГРЧЦ + время | вкл, 10:00 |
| Автоимпорт RIR+IANA + время | вкл, 06:00 |
| Пропускать неизменённый датасет ГРЧЦ | выкл |
| Кэш ZIP ГРЧЦ | вкл |

Workers `geoip_import` / `geoip_rir` перечитывают расписание при сохранении config.

### RIR registry (delegated-extended)

Отдельный слой (не ГРЧЦ). На **Обзоре** (2×2): **Проверить ГРЧЦ** / **Импортировать ГРЧЦ**, **Проверить RIR+IANA** / **Импортировать RIR+IANA**.  
Также Admin → **RIR registry** — статус снимка и кнопки импорта/сброса; ссылки на Browse.

| API | Описание |
|-----|----------|
| `GET /api/v1/admin/rir/status` | `rir_dataset_state` (+ `snapshotsByRegistry`) + активный run |
| `POST /api/v1/admin/rir/test` | probe 6 HTTPS latest-файлов (без записи в БД) |
| `POST /api/v1/admin/rir/imports/trigger` | очередь импорта (409 если уже running/queued) |
| `POST /api/v1/admin/rir/imports/reset` | сброс зависших queued/running |

**Browse UI:** `/browse/rir` (RIR IP, locked ipv4+ipv6) и `/browse/rir-asn` (RIR ASN, locked asn).  
**Dashboard:** три карточки — «Реестры RIR / IANA», «Активный датасет RIR / IANA», «Объем данных…» со ссылкой на расхождения ГРЧЦ ≠ RIR CC (`GET /api/v1/rir/status` + `GET /api/v1/table/cc-mismatch/state`).  
**Header:** hover-меню дат ГРЧЦ + RIPE/ARIN/APNIC/AFRINIC/LACNIC/IANA.

После **успешного** импорта RIR (как и после импорта ГРЧЦ) API запускает пересчёт `geo_rir_cc_mismatches` (soft-fail: ошибка rebuild не валит импорт). При старте API — backfill, если статус `never`/`failed`/stale `running` и оба датасета ready.

Логи: `docker compose logs rir`

### Мониторинг

- Dashboard → «Последние импорты» (шаги, ошибки, wall time)
- Portainer / CLI: `docker compose logs import`

### Проверка ГРЧЦ

Admin → **Обзор** → **Проверить ГРЧЦ**

API: `POST /api/v1/admin/config/test/grchc`

| Ответ | Значение |
|-------|----------|
| 200 `{ ok, downloadCount, latestDate }` | Creds верны |
| 400 `MissingCredentials` | Email/пароль не заданы |
| 502 `GrchcProbeFailed` | Ошибка ЛК |

### Проверка RIR+IANA

Admin → **Обзор** → **Проверить RIR+IANA**

API: `POST /api/v1/admin/rir/test`

| Ответ | Значение |
|-------|----------|
| 200 `{ ok, reachableCount, sources[] }` | Все 6 файлов доступны и парсятся |
| 502 `RirProbeFailed` | Хотя бы один источник недоступен / пуст |

### Восстановление данных

Автобэкап PostgreSQL **не используется**. При проблеме — **повторный import** из ГРЧЦ и/или RIR+IANA.

### Полный wipe датасетов

Admin → **Обзор** (или соответствующая кнопка wipe в UI) → `POST /api/v1/admin/data/wipe` (admin session).

Деструктивная операция: очищает таблицы ГРЧЦ и RIR, history import/export, ZIP cache, `rir_rdap_cache`, **`geo_rir_cc_mismatches`**, сбрасывает `geo_rir_cc_mismatch_state` в `never`, пересоздаёт пустые MV. Берёт тот же advisory lock, что и job пересчёта расхождений (сериализация с in-flight rebuild). Config/secrets и схема БД сохраняются.

После wipe нужны повторные импорты ГРЧЦ и RIR; сравнение CC построится автоматически, когда оба слоя снова ready.

## Reload и перезапуск

После save некоторые поля требуют restart:

- **API restart:** pool size, statement timeout, log level, rate limits и др.
- **Web reload:** обновление `proxy.env` после смены External key

Admin → **Обзор** показывает `pendingReload` hints.

API: `GET /api/v1/admin/config/reload-status` → `{ configUpdatedAt, pendingReload }`.
