# Безопасность

## Что является секретом

| Данные | Где хранить |
|--------|-------------|
| ГРЧЦ email/password | Admin → ГРЧЦ (encrypted `secrets.enc`) |
| Google Maps API key | Admin → Интеграции |
| API keys (import/proxy) | Admin → API (auto-generated on first boot) |
| Admin password | Admin → Admin доступ (scrypt hash) |
| `CONFIG_MASTER_KEY` | Env / Portainer (не в git) |
| `POSTGRES_PASSWORD` | Env / compose (не в git для prod) |

## CONFIG_MASTER_KEY

64 hex-символа. Шифрует `secrets.enc` в volume `config_data`. Задайте один раз в production и сохраните backup — без ключа secrets не расшифровать после redeploy.

## Что не коммитить

- `.env`, `.env.local`, любые `.env.*` кроме `.env.example`
- `data/`, `packages/api/data/` (локальный config store)
- `secrets.enc`, `proxy.env`, `meta.json`
- `backups/`, `*.sql.gz`
- Реальные API keys, пароли, email ГРЧЦ

Локальный `data/config/` после старой модели (env migration, `change-me-in-production` в `proxy.env`) лучше удалить и перезапустить API — создастся fresh config. Docker volume `config_data` не удаляйте без backup, если Admin уже настроен.

Проверка: `./scripts/check-public-ready.sh`

## Perimeter

- **NPM Access List** — кто может открыть UI
- **Admin login** — session cookie для `/admin`
- **API auth (production)** — `X-API-Key` на data-plane; nginx injects key server-side

## PgBouncer

Dev default `geoip`/`geoip` в `infra/pgbouncer/userlist.txt` допустим для open source. В production смените пароль и синхронизируйте userlist.

## История git

Если `.env` когда-либо коммитился:

```bash
git log --all --oneline -- .env
```

Рассмотрите ротацию ключей и `git filter-repo` перед public release.

## Docker

Prod overlay не публикует Postgres и API на хост — только web `:8080` за NPM.
