# Быстрый старт

Цель: за один проход развернуть GeoIP Analytics и выполнить первый IP lookup.

**Репозиторий:** [github.com/finenumbers/geoip](https://github.com/finenumbers/geoip)  
**Разработчик:** [Finenumbers](https://finenumbers.com) · apps@finenumbers.com

---

## 1. Развернуть stack

Выберите один способ — подробности в [РАЗВЁРТЫВАНИЕ.md](РАЗВЁРТЫВАНИЕ.md).

| Способ | Когда использовать |
|--------|-------------------|
| **Portainer** (рекомендуется) | Production на сервере — compose path `docker-compose.portainer.yml` |
| **Docker Compose (CLI)** | SSH, локальный prod-like стенд |

После деплоя откройте в браузере: `http://<хост>:8080`

## 2. Проверить, что сервис жив

```bash
curl -s http://<хост>:8080/api/v1/health
# {"status":"ok","timestamp":"..."}

curl -s http://<хост>:8080/api/v1/ready
# status: "not_ready" — нормально на пустой БД
```

На Dashboard (`/`) отображается checklist первичной настройки.

## 3. Пройти onboarding (5 шагов)

Шаги совпадают с `GET /api/v1/public/setup-checklist` и баннером на Dashboard.

| # | Шаг | Где | Обязательно |
|---|-----|-----|-------------|
| 1 | Создать учётную запись администратора | `/admin/setup` | да |
| 2 | Сгенерировать API-ключ **External IP Lookup** | `/admin/setup-api-key` | **да** |
| 3 | Указать учётные данные ЛК ГРЧЦ | Admin → **ГРЧЦ / Import** | да |
| 4 | Импортировать датасет GeoIP | Admin → **Обзор** → «Импортировать датасет» | да |
| 5 | Настроить Google Maps | Admin → **Интеграции** | нет |

### Шаг 1 — Admin

1. Откройте `http://<хост>:8080/admin/setup`
2. Задайте логин и пароль
3. После сохранения страница setup больше недоступна — вход через `/admin/login`

### Шаг 2 — API-ключ External IP Lookup (обязательно)

1. После setup откроется `/admin/setup-api-key`
2. Нажмите **Generate**, скопируйте ключ (после save он маскируется)
3. Сохраните — без этого ключа data-plane API (`lookup`, таблицы, export) вернёт **401**

> Ключ **не** создаётся автоматически вместе с admin. Это отдельный обязательный шаг.

### Шаг 3 — ГРЧЦ

1. Admin → **ГРЧЦ / Import**
2. Email и пароль личного кабинета GeoIP на сайте ГРЧЦ (у каждого оператора свои)
3. **Сохранить** → **Проверить ГРЧЦ** (должно быть успешно)

### Шаг 4 — Import

1. Admin → **Обзор** → **Импортировать датасет**
2. Следите за прогрессом на Dashboard (таблица «Последние импорты»)
3. Полный датасет — **десятки минут и более**; не останавливайте контейнер `geoip_import`

Альтернатива: дождаться cron автоимпорта (Admin → Общие; default ежедневно 10:00 в выбранном TZ) после сохранения creds ГРЧЦ.

> **Не используйте** `seed:fixture` в production — только для локальной разработки и CI.

### Шаг 5 — Google Maps (опционально)

Admin → **Интеграции** → ключ Maps Embed API. Нужен для карты на странице Lookup.

## 4. Понять состояние `/api/v1/ready`

| status | HTTP | Значение |
|--------|------|----------|
| `not_ready` | 503 | БД пустая, MV не готовы или import не завершён |
| `degraded` | 503 | Основные данные есть; ASN mapping ещё догружается или идёт import |
| `ready` | 200 | Все проверки пройдены |

Проверки в ответе:

| check | true когда |
|-------|------------|
| `database` | PostgreSQL доступен |
| `dataset` | Есть активный датасет |
| `materializedViews` | MV готовы |
| `productionIndexes` | Production-индексы на месте |
| `asnMapping` | ASN mapping готов |
| `importRunning` | Сейчас выполняется import |

В состоянии `degraded` browse и lookup уже могут работать — смотрите Dashboard.

```bash
curl -s http://<хост>:8080/api/v1/ready | jq .
```

## 5. Первый lookup

### Через UI

1. Откройте `/lookup`
2. Введите IP (например `8.8.8.8`) → **Проверить**
3. Включите секции City / Country / ASN / Map

### Через API

```bash
curl -sS -X POST 'http://<хост>:8080/api/v1/lookup' \
  -H 'Content-Type: application/json' \
  -H 'X-API-Key: ВАШ_КЛЮЧ_ИЗ_SETUP' \
  -d '{"ip":"8.8.8.8","include":["city","country","asn"]}'
```

Подробнее: [ПЕРИМЕТР-И-HTTPS.md](ПЕРИМЕТР-И-HTTPS.md), [СПРАВОЧНИК-API.md](СПРАВОЧНИК-API.md), страница `/api-docs`.
