## Локальная разработка

### Требования

- Node.js 20+
- pnpm 9+
- Docker (для Postgres)

### Запуск

```bash
cp .env.example .env
docker compose up postgres -d
pnpm install
pnpm db:migrate
pnpm --filter @geoip/api seed:fixture   # только dev/CI — тестовый датасет
pnpm dev
```

| Сервис | URL |
|--------|-----|
| API | http://localhost:3000 |
| Web | http://localhost:5173 |

При первом запуске API создаёт config store в `CONFIG_DATA_DIR` (default `./data/config`). ГРЧЦ и Google Maps — пустые, заполните в `/admin`.

> **`seed:fixture`** — только dev/CI. **Не** используйте в production.

### Структура monorepo

```
packages/
  shared/   # Zod schemas, defaults, types
  api/      # Fastify backend, workers
  web/      # React SPA
fixtures/   # Sample CSV (tests only)
docs/       # Документация (RU)
```

Миграции БД: `packages/api/migrations/*.sql` — единственный источник (не drizzle-kit generate).

```bash
chmod +x scripts/check-public-ready.sh
pnpm knip
./scripts/check-public-ready.sh
pnpm lint && pnpm test
```

`check-public-ready.sh` — secrets / config artifacts не в git.

Integration tests требуют Postgres (CI поднимает service; локально — `docker compose up postgres -d`).

## Безопасность

### Что является секретом

| Данные | Где хранить |
|--------|-------------|
| ГРЧЦ email/password | Admin → ГРЧЦ (`secrets.enc`) |
| Google Maps API key | Admin → Интеграции |
| External IP Lookup key | Admin → API / `/admin/setup-api-key` |
| Import API key | Admin → API (reserved, не в HTTP) |
| Admin password | Admin → Admin доступ (scrypt hash) |
| `CONFIG_MASTER_KEY` | Env / Portainer (не в git) |
| `POSTGRES_PASSWORD` | Compose (не default `geoip` в prod) |

### CONFIG_MASTER_KEY

64 hex. Шифрует `secrets.enc`. Задайте в production и сохраните backup.

Если не задан — auto-gen в `.master-key` (mode 0600) в volume. Включите в backup `config_data`.

### Что не коммитить

- `.env`, `.env.local`, любые `.env.*` кроме `.env.example`
- `data/`, `packages/api/data/`
- `secrets.enc`, `proxy.env`, `meta.json`, `.master-key`
- `backups/`, `*.sql.gz`
- Реальные API keys, пароли, email ГРЧЦ

Локальный `data/config/` с повреждёнными файлами — удалите и перезапустите API (fresh store). Docker volume `config_data` не удаляйте без backup.

### Perimeter

- NPM Access List — кто открывает UI
- Admin login — session cookie
- Data API — `X-API-Key` (`authEnabled=true` в prod)
- SPA — nginx injects key из `proxy.env`
- External lookup — клиентский key + NPM `X-GeoIP-Client-Auth: 1`

См. [ПЕРИМЕТР-И-HTTPS.md](ПЕРИМЕТР-И-HTTPS.md).

### PgBouncer

Dev default `geoip`/`geoip` допустим для open source. В production смените пароль и синхронизируйте userlist.

### Admin rate limit

5 попыток / 15 мин на IP — **in-memory** (сброс при restart API). См. [ROADMAP.md](ROADMAP.md) §1.

### Production hardening

- Generic 500 errors (без stack trace)
- Timing-safe API key compare
- UUID validation на export `:id`
- GRChC `filedate` validation `\d{8}`

## Устранение неполадок

### RIR browse 503 `RirNotReady`

Снимок delegated ещё не загружен. Проверьте `docker compose logs rir`, Admin → RIR registry → trigger, или дождитесь cron (Admin → Общие).

### Все `geoip_*` в статусе Exited (не crash loop)

Если Postgres в логах пишет `received fast shutdown request` / `terminating connection due to administrator command`, а nginx — graceful `exit` в тот же момент — stack **остановили** (Portainer Stop / `docker compose down` / `docker stop`), а не уронили import/OOM.

1. Portainer → Stacks → geoip → **Start** (или `docker start` контейнеров)
2. Без `restart: unless-stopped` контейнеры после stop остаются exited — политика задана в `docker-compose.portainer.yml` / `docker-compose.yml` (нужен redeploy после обновления compose)
3. Exit code **1** у `geoip_api` при общем stop часто из‑за обрыва idle-соединений к Postgres; liveness смотрите по `/api/v1/health` после старта

### `/ready` = `not_ready` после установки

**Ожидаемо** на пустой БД. Checklist: Admin → ГРЧЦ → Import → дождитесь MV.

```bash
curl -s http://localhost:8080/api/v1/ready | jq .
```

### `/ready` = `degraded`

Core data ready; ASN mapping догружается или import running. Browse/lookup могут работать.

### Import failed

1. Admin → **Проверить ГРЧЦ**
2. Dashboard → последние импорты → детали шага с ошибкой
3. Logs: `docker compose logs import`

Cron **не создаёт** failed run без creds — import пропускается.

### Portainer: `lstat .../packages: no such file`

Compose path должен быть **`docker-compose.portainer.yml`**, не `docker-compose.yml`. Pull and redeploy.

### Portainer: `userlist.txt` mount error

Bind mount `./infra/` недоступен — используйте `docker-compose.portainer.yml` (inline configs).

### Stack «Limited»

Пересоздайте stack через Portainer Repository. См. [РАЗВЁРТЫВАНИЕ.md](РАЗВЁРТЫВАНИЕ.md#stack-limited-created-outside-of-portainer).

### Admin не открывается / 401

1. `/admin/setup` при первом запуске
2. NPM Access List
3. Session cookie — same-origin через ваш домен

### API 401 на Dashboard

`authEnabled=true`. Nginx читает `proxy.env`. Redeploy web если volume пуст при старте.

### External lookup 401

Custom NPM location + `X-GeoIP-Client-Auth: 1`; ключ из Admin → API.

### Карта на Lookup не работает

1. Admin → Интеграции → Google Maps key
2. HTTP referrer restriction в GCP
3. Runtime pick-up — rebuild web не нужен

### Export CSV

Кнопка на Browse выгружает **всю** фильтрованную выборку (до Max rows). ZIP с CSV внутри. Worker logs: `docker compose logs export`.

### Где API keys после fresh install?

- **External IP Lookup key** — явно на `/admin/setup-api-key` (обязательно)
- Import API key — автоген, masked в Admin (не для HTTP)
- `proxy.env` — обновляется при save External key

### Нужен ли `.env` в Docker?

Нет для базового деплоя. Опционально для CLI interpolation. Секреты — Admin UI.

### PgBouncer auth failed после смены пароля

Синхронизируйте `POSTGRES_PASSWORD` в postgres, `x-api-database-env` и `pgbouncer_userlist`.

### Config lock / stale lock

При concurrent write config store — retry. Stale lock логируется API. Restart API если зависло.
