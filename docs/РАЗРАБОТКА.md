# Разработка

Руководство для contributors: локальный запуск, fixtures, тесты.

## Требования

- Node.js 20+
- pnpm 9+
- Docker (для Postgres)

## Локальный запуск

```bash
cp .env.example .env
docker compose up postgres -d
pnpm install
pnpm db:migrate
pnpm --filter @geoip/api seed:fixture   # только dev/CI — наполняет БД тестовым датасетом
pnpm dev
```

- API: http://localhost:3000
- Web: http://localhost:5173

При первом запуске API создаёт config store в `CONFIG_DATA_DIR` (по умолчанию `./data/config`) с автогенерированными API keys. ГРЧЦ и Google Maps остаются пустыми — заполните в `/admin`.

## Скрипты

| Команда | Назначение |
|---------|------------|
| `pnpm dev` | API + web + workers |
| `pnpm test` | Unit/integration tests |
| `pnpm db:migrate` | Drizzle migrations |
| `pnpm --filter @geoip/api seed:fixture` | Fixture dataset (не для production) |
| `pnpm --filter @geoip/api import:trigger` | Импорт датасета из CLI (нужны creds ГРЧЦ в Admin) |
| `pnpm --filter @geoip/api probe:lk` | Probe ЛК ГРЧЦ |

## Тесты

```bash
pnpm test
```

CI использует `seed:fixture` для smoke/integration tests — см. [`.github/workflows/ci.yml`](../.github/workflows/ci.yml).

## Структура monorepo

```
packages/
  shared/   # Zod schemas, defaults, types
  api/      # Fastify backend, workers
  web/      # React SPA
fixtures/   # Sample CSV (tests only)
docs/       # Документация (RU)
```

## Public release checklist

Перед push в public GitHub:

```bash
chmod +x scripts/check-public-ready.sh
./scripts/check-public-ready.sh
```

Не коммитьте `.env`, `data/`, `packages/api/data/`, `backups/`, реальные ключи ГРЧЦ или Google Maps.

Репозиторий: https://github.com/finenumbers/geoip
