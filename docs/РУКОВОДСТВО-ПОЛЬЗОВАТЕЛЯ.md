## Dashboard

**URL:** `/`

### Баннеры

- **Setup checklist** — если onboarding не завершён (ссылки на шаги)
- **System initializing** — MV ещё готовятся
- **Query error** — ошибка загрузки данных

### Карточки

**System status** — `ready` / `degraded` / `not_ready` / initializing:

| Check | Значение |
|-------|----------|
| database | PostgreSQL доступен |
| dataset | Есть активный датасет |
| materializedViews | MV готовы |
| productionIndexes | Production-индексы |
| asnMapping | ASN mapping готов |

**Active dataset** — дата датасета, activated at, fingerprint, next import, server time.

**Data volume** — размер БД; counts: city blocks, city locations, RU city blocks, ASN blocks, IPv4 addresses.

### Последние импорты

Таблица: ID, date, status, trigger, wall time, started. Раскрытие строки — детали шагов (status, duration, rows, messages, error).

### Import benchmark

Wall time последнего успешного import + breakdown по шагам.

Данные обновляются автоматически (polling).

## Lookup (IP Lookup)

**URL:** `/lookup`  
**Deep link:** `/lookup?ip=8.8.8.8`

### Как пользоваться

1. Введите IPv4 или IPv6
2. Нажмите **Проверить**
3. Выберите секции ответа (минимум одна):
   - **City** — город, регион, координаты
   - **Country** — страна
   - **ASN** — автonomous system
   - **Map** — карта (требует Google Maps key)

### Ответ

Карточки key/value по выбранным секциям. Если IP не найден — «not found». Показываются `datasetDate` и queried IP.

### Карта

Если ключ Google Maps не настроен — ссылка на Admin → Интеграции. Ключ подхватывается runtime без пересборки web.

## Browse (Таблица)

**URL:** `/browse/city` — city blocks  
**URL:** `/browse/country` — country blocks

Переключение вкладок сбрасывает sort и filters.

### Возможности

- **Сортировка** по колонкам (сохраняется в URL)
- **Фильтры** — text и facet (multi-select)
- **Infinite scroll** — подгрузка по 100 строк (до 5000 loaded)
- **Virtualized table** — счётчики loaded / total rows
- **Active filter chips** — удаление отдельных фильтров

### Колонки и фильтры

| Колонка | Фильтр | City | Country |
|---------|--------|------|---------|
| Network | Text | ✓ | ✓ |
| Prefix | Text (validated) | ✓ | ✓ |
| Country ISO | Text | ✓ | ✓ |
| Country name | Facet (multi) | ✓ | ✓ |
| Region (`subdivision_1_name`) | Facet | ✓ | — |
| City name | Facet | ✓ | — |
| ASN | Text (numeric) | ✓ | ✓ |
| ASN Org | Facet (limit 200) | ✓ | ✓ |

Country tab **не** поддерживает region и city name (ограничение исходника ГРЧЦ).

### Предупреждения

- Slow sort — полная сортировка по тяжёлым полям
- RU partial sort override
- Offset-only pagination cap
- Estimated row count при ASN filter

### Сброс

**Reset filters** — очистка всех фильтров и sort.

## Export CSV

Export доступен **на странице Browse** (отдельной страницы нет).

### Как экспортировать

1. Задайте фильтры на `/browse/city` или `/browse/country`
2. Нажмите **Export CSV**
3. Дождитесь завершения job (polling)
4. Скачается **ZIP** с CSV внутри:
   - `geoip-city-export.csv` или
   - `geoip-country-export.csv`

Export выгружает **всю** отфильтрованную выборку, не только строки на экране.

### Лимиты

- Max rows — Admin → Export → Max rows (default 5 000 000)
- При превышении — ошибка с estimated count
- При 503 — MV refreshing, повторите позже

Формат CSV: UTF-8 BOM, разделитель `;`.

## API Docs

**URL:** `/api-docs`

Встроенная справка по **POST /api/v1/lookup**:

- Auth: заголовок `X-API-Key`
- Request: `ip`, optional `include` (`city`, `country`, `asn`)
- Примеры: curl, JavaScript fetch, Python requests
- Коды ошибок: 401, 400, 422, 429

Полный список endpoint'ов: [СПРАВОЧНИК-API.md](СПРАВОЧНИК-API.md).  
NPM и внешний доступ: [ПЕРИМЕТР-И-HTTPS.md](ПЕРИМЕТР-И-HTTPS.md).

## Доступность до import

| Состояние `/ready` | Что видит пользователь |
|--------------------|------------------------|
| `not_ready` | Checklist на Dashboard; Browse/Lookup могут быть недоступны или пусты; `/ready` = 503 |
| `degraded` | Данные частично готовы; ASN mapping или import in progress |
| `ready` | Полный функционал |

Пока import не завершён — `/api/v1/ready` возвращает 503. Это нормально после установки.
