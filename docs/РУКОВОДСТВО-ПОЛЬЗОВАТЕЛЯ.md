## Dashboard

**URL:** `/`

### Баннеры

- **Setup checklist** — если onboarding не завершён (ссылки на шаги)
- **System initializing** — MV ГРЧЦ ещё готовятся
- **Query error** — ошибка загрузки данных

### Карточки ГРЧЦ

**Подсистема ГРЧЦ** — `ready` / `degraded` / `not_ready` / initializing:

| Check | Значение |
|-------|----------|
| database | PostgreSQL доступен |
| dataset | Есть активный датасет |
| materializedViews | MV готовы |
| productionIndexes | Production-индексы |
| asnMapping | ASN mapping готов |

**Активный датасет ГРЧЦ** — дата датасета, activated at, fingerprint, next import, server time.

**Объем данных датасета ГРЧЦ** — размер БД; counts: city blocks, city locations, RU city blocks, ASN blocks, IPv4 addresses.

### Карточки RIR / IANA

Три отдельные карточки (данные из `GET /api/v1/rir/status`). На карточках **нет** кнопки «Открыть таблицу» и блока «По status» — таблицы открываются из меню **Таблица → RIR IP / RIR ASN**.

1. **Реестры RIR / IANA** — общий счётчик диапазонов (или «Снимок ещё не загружен»); по каждому registry — число строк и дата снимка.
2. **Активный датасет RIR / IANA** — дата последнего снимка, статус импорта, activated / next import, server time, last error.
3. **Объем данных датасета RIR / IANA** — размер таблицы; всего диапазонов; уникальные IPv4; ссылка **Расхождения ГРЧЦ ≠ RIR CC** → `/cc-mismatch` (значение: число при `ready`, «Пересчёт…», «Ошибка» или «—»); строка **Время расчета расхождений**.

История снимков RIR и «трансферы» в продукте **не показываются** (слой удалён).

### Последние импорты

Таблица: плоскость (**ГРЧЦ** / **RIR+IANA**), ID, date, status, trigger, wall time, started. Раскрытие строки — детали шагов.

### Import benchmark

Wall time последнего успешного import ГРЧЦ + breakdown по шагам.

Данные обновляются автоматически (polling). Пока сравнение CC в статусе `running`, блок расхождений на Dashboard опрашивается чаще (~5 с).

## Lookup (IP Lookup)

**URL:** `/lookup`  
**Deep link:** `/lookup?ip=8.8.8.8`

Lookup работает по датасету **ГРЧЦ** (longest-prefix match). Слой RIR в Lookup не участвует.

### Как пользоваться

1. Введите IPv4 или IPv6
2. Нажмите **Проверить**
3. Выберите секции ответа (минимум одна):
   - **City** — город, регион, координаты
   - **Country** — страна
   - **ASN** — autonomous system
   - **Map** — карта (требует Google Maps key)

### Ответ

Карточки key/value по выбранным секциям. Если IP не найден — «not found». Показываются `datasetDate` и queried IP.

### Карта

Если ключ Google Maps не настроен — ссылка на Admin → Интеграции. Ключ подхватывается runtime без пересборки web.

## Browse (Таблица)

Пять вкладок:

| Вкладка | URL | Источник данных |
|---------|-----|-----------------|
| City blocks | `/browse/city` | MV city (ГРЧЦ) |
| Country blocks | `/browse/country` | MV country (ГРЧЦ) |
| ASN blocks | `/browse/asn` | ASN mapping ГРЧЦ |
| RIR IP | `/browse/rir` | `rir_delegations`, только ipv4/ipv6 |
| RIR ASN | `/browse/rir-asn` | `rir_delegations`, только asn |

Переключение вкладок **сбрасывает** sort и filters к значениям по умолчанию для этой вкладки.

### Общие возможности

- **Сортировка** по колонкам (сохраняется в URL)
- **Фильтры** — text и facet (multi-select) в заголовках колонок
- **Активные фильтры** — горизонтальная полоса chips («Активные фильтры:») с отменой отдельных значений
- **Infinite scroll** — подгрузка по 100 строк (до 5000 loaded на экране)
- **Virtualized table** — счётчики loaded / total rows
- **Экспорт CSV** / **Сбросить фильтры**

Sort и filters хранятся в query string URL (`sort`, `filters` как JSON).

### ГРЧЦ: City / Country / ASN

| Колонка | Фильтр | City | Country | ASN blocks |
|---------|--------|------|---------|------------|
| Network | Text (`startsWith`) | ✓ | ✓ | ✓ |
| Prefix | Text (validated) | ✓ | ✓ | — |
| Country ISO | Text | ✓ | ✓ | — |
| Country name | Facet (multi) | ✓ | ✓ | — |
| Region | Facet | ✓ | — | — |
| City name | Facet | ✓ | — | — |
| ASN | Text | ✓ | ✓ | ✓ |
| ASN Org | Facet | ✓ | ✓ | ✓ |

Country tab **не** поддерживает region и city name (ограничение исходника ГРЧЦ).

### RIR IP (`/browse/rir`)

Заблокированный фильтр: `resource_type ∈ {ipv4, ipv6}`. Его нельзя снять полностью (chip `resource_type` в полосе активных фильтров **скрыт**). Facet **Type** показывает только `ipv4` / `ipv6` (без `asn`); можно сузить до одного семейства — при очистке снова оба.

Типичные колонки: Range, Type, Prefix, Family, Hosts, Network, ISO (`cc`), Status, Date, Opaque ID, RIR (`registry`).

Семантика `cc` в RIR — **legal country of holder** (delegated), не геолокация ГРЧЦ.

### RIR ASN (`/browse/rir-asn`)

Заблокированный фильтр: `resource_type = asn` (всегда). Колонки: Range, ASN start, ASN count, ISO, Status, Date, Opaque ID, RIR — без Type / Prefix / Family / Hosts / Network.

### Детали строки RIR (RDAP / PeeringDB)

Клик по строке на вкладках **RIR IP** или **RIR ASN** открывает модал **Детали диапазона RIR**:

- блок Delegated (поля снимка);
- **RDAP** — обогащение (Handle, Name, Status, Country, Entities);
- **PeeringDB** — только для `resource_type = asn`; иначе сообщение, что записи нет.

### Неготовность RIR

Если снимок RIR ещё не загружен, Browse RIR отвечает **503** (`RirNotReady`). В UI — сообщение вроде «Снимок RIR ещё не загружен…». Готовность GeoIP `/ready` от RIR **не зависит**.

### Предупреждения (в основном ГРЧЦ)

- Slow sort — полная сортировка по тяжёлым полям
- RU partial sort override
- Offset-only pagination cap
- Estimated row count при ASN filter

### Сброс

**Сбросить фильтры** — очистка фильтров и sort (для RIR снова выставляется locked `resource_type`).

## ГРЧЦ ≠ RIR (расхождения CC)

**URL:** `/cc-mismatch`  
**Пункт меню:** ГРЧЦ ≠ RIR  
**Также:** ссылка с карточки объёма RIR на Dashboard

### Зачем нужна страница

Полное сравнение **country blocks ГРЧЦ** с **покрывающим** CIDR из снимка RIR (ipv4/ipv6). В таблице остаются только строки, где ISO страны ГРЧЦ **не совпадает** (без учёта регистра) с CC делегирования RIR. Совпадения не показываются.

Это **отдельный материализованный слой**, не live-join в Browse. Lookup и `/ready` ГРЧЦ от него не зависят. CSV-экспорта на этой странице **нет**.

### Когда строится сравнение

Пересчёт запускается автоматически:

1. после **успешного** импорта ГРЧЦ;
2. после **успешного** импорта RIR+IANA;
3. при старте API, если статус `never` / `failed` или «зависший» `running` дольше ~30 минут — и оба датасета уже ready.

Если готов только один из датасетов, пересчёт **не трогает** таблицу (нет ложного `ready` с нулём строк). Ошибка пересчёта **не откатывает** успешный импорт.

На странице **нет** кнопки «Пересчитать вручную» — достаточно дождаться автозапуска или следующего импорта.

### Статусы

| Статус | Баннер | Смысл |
|--------|--------|--------|
| `never` | да | Сравнение ещё не строилось |
| `running` | да | Идёт пересчёт (полные country blocks — может занять заметное время) |
| `failed` | да | Ошибка; в баннере — текст и длительность |
| `ready` | **нет** (тихий режим) | Таблица актуальна; при 0 строк — «Расхождений нет» |

Пока `never` или `running`, UI опрашивает статус примерно раз в 5 с; при `ready`/`failed` — реже.

### Колонки и фильтры

| Колонка | Поле API | Фильтр в UI |
|---------|----------|-------------|
| Network | `network` | Text |
| Range | `range_text` | Text (covering RIR CIDR) |
| ASN | `asn` | Text |
| ASN Org | `asn_org` | Facet (multi) |
| ГРЧЦ | `grchc_cc` | Text (ISO из ГРЧЦ) |
| RIR | `rir_cc` | Text (CC из RIR) |
| Registry | `registry` | Text (ripe/arin/…/iana) |

Все колонки сортируются. Над таблицей — та же полоса **Активные фильтры**, что и в Browse: chips с отменой. Locked-фильтров здесь нет.

ASN / ASN Org берутся из mapping country-block ↔ ASN на стороне ГРЧЦ (если mapping ещё не готов, поля могут быть пустыми — «—»).

### Пагинация

Infinite scroll по 100 строк, до 5000 на экране. При одноколоночной сортировке — keyset (для ASN — числовой курсор); при сложной сортировке — offset с ограничением.

## Export CSV

Export доступен **на странице Browse** (отдельной страницы нет). Страница `/cc-mismatch` экспорт **не** предлагает.

### Как экспортировать

1. Задайте фильтры на нужной вкладке Browse
2. Нажмите **Экспорт CSV**
3. Дождитесь завершения job (polling)
4. Скачается **ZIP** с CSV внутри:

| Вкладка | Имя файла в ZIP |
|---------|-----------------|
| City | `geoip-city-export.csv` |
| Country | `geoip-country-export.csv` |
| ASN blocks | `geoip-asn-export.csv` |
| RIR IP / RIR ASN | `geoip-rir-export.csv` |

Export выгружает **всю** отфильтрованную выборку, не только строки на экране. Для RIR в запрос всегда уходит locked `resource_type` текущей вкладки (IP ≠ ASN).

### Готовность и лимиты

- **City / Country / ASN** — нужны готовые MV ГРЧЦ; иначе **503** («Materialized views are refreshing…»)
- **RIR** — нужен ready-снимок RIR; иначе **503** (даже если MV ГРЧЦ ещё не готовы — RIR-экспорт от MV не зависит)
- Max rows — Admin → Export → Max rows (default 5 000 000)
- При превышении — ошибка с estimated count

Формат CSV: UTF-8 BOM, разделитель `;`.

## Доступность до import

| Состояние `/ready` (ГРЧЦ) | Что видит пользователь |
|---------------------------|------------------------|
| `not_ready` | Checklist на Dashboard; Browse/Lookup ГРЧЦ могут быть недоступны или пусты; `/ready` = 503 |
| `degraded` | Данные частично готовы; ASN mapping или import in progress |
| `ready` | Полный функционал ГРЧЦ |

Пока import ГРЧЦ не завершён — `/api/v1/ready` возвращает 503. Это нормально после установки.

RIR и страница расхождений живут **отдельно**:

- без снимка RIR — вкладки RIR IP / RIR ASN недоступны (503), карточки RIR на Dashboard показывают «ещё не загружен»;
- без обоих ready-датасетов или до конца пересчёта — `/cc-mismatch` в статусе `never` / `running`, число расхождений на Dashboard — «—» или «Пересчёт…».
